Expression Sources

Below is a table describing the various expression sources.

Source control Grabbed expression becomes Rationale
binding node A variable (or application of variable) corresponding to the binding. Instantiation of variables.
if-then button
 A new if-then-else expression with undefined condition, consequent and alternetive. Creation of conditional expressions.
let button
 A new let-expression, with an undefined body and no variable bindings. Creation of local definitions.
apply button
 A new function application (user is prompted for the number of arguments). The function and arguments are undefined. Allows application of higher-order function arguments; explicit expression of function application.
lambda button
 A new lambda abstration (user is prompted for the number of arguments). The body is undefined, the bindings are variable bindings with default names. Allows creation of anonymous function-valued expressions.
datatype button
 A new algebraic datatype. The body (scope) of the expression is undefined, and the datatype has no constructors. Allows creation of new algebraic data type.
list button
 A new empty list. Allows creation of a list.
patterns button
 A new pattern-set expression (user is prompted for the number of arguments). The new expression has a single pattern, with variable bindings and an undefined body. Allows creation of pattern-matching function definitions.
load button
 A new expression loaded from a file. Allows expressions and programs to persist on disk.
primitive button
 A new built-in function (or application of built-in function). The user is prompted for the function. Allows access to built-in functions and constants.
guards button
 A new guarded expression set. New expression has one guard-value pair, guard and value are undefined. Allows creation of guarded expressions.
thread button
 A new interpreter control point. Allows control of interpreter parameters and single-step evaluation.
literal field
 A new literal expression. Integers, floats, characters and strings are supported. Allows creation of basic data items.
primitive function buttons A new built-in function (or application of built-in function). Provides quick access to common primitives.
cut control The subexpression that the control panel was summoned for. Allows movement and deletion of subexpressions.
copy control A copy of the subexpression that the control panel was summoned for. Allows duplication of subexpressions, useful for saving and writing expressions.
make variable control A varible corresponding to the subexpression that the control panel was summoned for. Instantiation of variables.
binding name label on let node A variable (or application of variable) corresponding to the binding. Instantiation of variables.

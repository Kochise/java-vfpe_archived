As we suspected, the generation of the left sub - list actually forced the generation of the right sub - list.

What happened ? Well, in order to fully generate the left side, the partition function had to examine the entire input list (because there might have been a left - side element at the very end). The way the partition function is written, it also generates the right side list on the way through. So under a "second evaluation waits for first evaluation" strategy, there would be no advantage in parallelising here.

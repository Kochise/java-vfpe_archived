The mergesort evaluation required 467 reduction steps to sort the list. Note that the comparison here between the reduction count for quicksort and mergesort is not really indicitive of the relative efficiencies of the algorithms. The quicksort algorithm has a fast in - place imperative implementationm, but that is not a factor in this functional implementation.

In fact, the profile information provided by the VFPE could be used to compare different implementation variations for these sorting algorithms, to find more efficient functional implementations, but that is beyond the scope of this example.
